    • Boring listing of HTML5/CSS3/etc skills doesn't provide a complete picture of developer.

      That's why I will describe my experience, technologies I use and learned moral principles.
      HTML
      I prefer to use Pug as template engine but I also worked with Haml.
      I had to do a lot of layout bugfixing and I realized that we still must strive to teach developers make better html layout and css styles for the overall good.
      CSS
      I use SCSS due to its popularity but I also worked with SASS, Less, Stylus, SSS.
      I use normalize.
      I use Autoprefixer and other css size optimization plugins at production build.
      I worked with Bootstrap, Foundation, UIkit. Using CSS framework is not a big deal, so I can use any you wish.
      I insist on using Stylelint everywhere.
      I use mix of SMACSS and BEM
      I am against too complex styles and prefer to use variables for colors, fonts, etc. and simple mixins
      I like mobile first approach and think that all sites must be responsive
      I fixed crossbrowser bugs for Chrome, Firefox (29+), Safari, ie9+, edge
      Hover effect on buttons with a bit of transition make websites so cozy and warm
      Boilerplate
      I use my own Gulp boilerplate to make such routine tasks ("Robots must suffer") as generation of iconfont, svg-sprite, retina sprites, css optimization, babel, image minifaction, stylelint etc.
      I worked with yarn/npm/bower
      JavaScript
      Examples of my code can be found at github
      Git
      I worked with Git following Gitflow process.
      In case of submodule structure or resolving messy merge conflict I prefer to use more visual tools like GitKraken, SourceTree, TortoiseGit.
      I had to use SVN at one project and after Git I don't like it at all.
      IDE
      I use Visual Studio Code or Sublime Text but also tried Atom, Brackets and WebStorm.
      Emmet forever
      Email
      I have experience making responsive email layout (with Foundation for emails or without any framework) for outlook 2007+, gmail (android), yandex, mail.ru, ios.

  • My name is Sergei Piadyshev, Front-end developer. I studied in Ulyanovsk State Technical University at "Publishing Business and Editing" faculty where I for the first time found out how websites are created. At that time I made my first simple table layout site about Harry Harrison.

    But somehow I became more interested in English that is why I decided to try to be a translator in the future. After university completion I worked as freelance translator for 0,5 year and then in a bank support center for 1 year. Then I found a job of technical translator in a local translation agency where I worked for 3 years.

    Finally I realized that I don't enjoy to be a translator anymore in addition to other reasons... and I decided to go back to creation of websites.

    January 2014

    I started to refresh my knowledge of HTML and CSS at HTML Academy and my acquaintance who had several years of experience as freelance html developer gave me part of his job, so I had extra part-time job at this period.

    August 2015 - April 2016

    I quit my job and then I was offered a position of HTML developer in an outsource company Kontora Ivanova.

    Continued learn new things at Codecademy.
    Helped to teach several new employees HTML and CSS.

    May 2016 - October 2016

    I began to work in Russian IT Group as HTML developer.

    Continued learn new things at freeCodeCamp.
    Implemented usage of Gulp boilerplate to make layout for Bitrix templates.
    Teached 2 students how to make HTML layout in upcoming 2017 as part of their internship in the company.

    October 2016 - Present

    I began to work in SimbirSoft as HTML developer.
    Started to learn JavaScript at Hexlet.
